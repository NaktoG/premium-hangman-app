# Contributing

## Branching

- `main`: producción estable.
- `develop`: integración.
- `feature/*`: nuevas funcionalidades.
- `fix/*`: correcciones.
- `chore/*`: mantenimiento.
- `docs/*`: documentación.
- `ci/*`: pipelines.

## Commits

Usar Conventional Commits:

```txt
feat(scope): add new behavior
fix(scope): correct bug
refactor(scope): improve structure without behavior change
test(scope): add or update tests
docs(scope): update documentation
ci(scope): update automation
chore(scope): maintenance task
```

## Pull Requests

Cada PR debe incluir:

- Contexto del cambio.
- Evidencia de pruebas.
- Impacto visual si aplica.
- Checklist de accesibilidad si toca UI.
